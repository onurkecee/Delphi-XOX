このリリース ノートでは、Delphi 10.1 Berlin、C++Builder 10.1 Berlin、RAD Studio 10.1 Berlin で作成するマルチデバイス アプリケーションに、カスタム スタイルを追加する手順の概要を説明します。

1. [マスタ] ビューを選択して、TStyleBook コンポーネントをフォームに追加します。

2. [マスタ] ビューで、ツールバーのドロップダウン メニューからマスタ スタイルを選択し、関連するプレミアム スタイルをスタイル パックから読み込みます。たとえば、マスタ スタイルに [Android] を選択している場合は、AndroidCoralCrystal.style ファイルを読み込んで [マスタ] ビューの StyleBook に割り当てます。カスタム スタイルを使用する場合、[マスタ] ビューを含むすべてのビューに、それぞれスタイルが必要です。

3. 作成したそれぞれのビューに切り替えて、そのビューの TStyleBook コンポーネントを選択し、そのプラットフォームのカスタム スタイルを読み込みます（[Windows デスクトップ] ビューなら Windows スタイル、[Android...] ビューなら Android スタイル、[OS X デスクトップ] ビューなら Mac スタイル、[iPad...] および [iPhone...] ビューなら iOS スタイル）。メモ： iPad と iPhone に異なるビューを作成している場合には、それぞれのビューに同じ iOS スタイルを読み込む必要があります。

4. アプリケーションに複数のフォームが含まれている場合、それぞれのビューで TStyleBook.UseStyleManager = True と設定すると、実行時には他のすべてのフォームで同じカスタム スタイルを使用することができます。TStyleBook.UseStyleManager = True と設定されている場合、アプリケーションに含まれているそのプラットフォーム向けの他のすべてのフォーム（Form2、Form3 など）で、システム スタイルがカスタム スタイルで完全に上書きされます。TStyleBook.UseStyleManager = False と設定されている場合、新しいフォーム（Form2、Form3 など）ではデフォルトのプラットフォーム スタイルが使用され、カスタマイズするには、Form2 の [マスタ] ビューに TStyleBook を追加して、アプリケーションに含まれる追加のフォームに作成されたすべてのビューに対してそれぞれカスタム スタイルを読み込み直さなければなりません。
